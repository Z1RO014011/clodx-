# Clodx Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a local-first macOS/Windows tray widget that reads and displays Codex quota windows, reset times, and available reset credits.

**Architecture:** Tauri 2 hosts a Rust provider layer and native tray/window behavior. React renders a compact quota widget from a normalized provider snapshot. The Rust provider reads the local Codex authentication state, queries the quota service with bounded requests, and returns explicit failure states.

**Tech Stack:** Tauri 2, Rust, React, TypeScript, Vite, Vitest.

---

## File structure

- `src-tauri/src/models.rs`: shared quota and preference models.
- `src-tauri/src/codex.rs`: local authentication loading, quota response parsing, and provider fetch.
- `src-tauri/src/lib.rs`: cache, Tauri commands, tray, and window lifecycle.
- `src/lib/format.ts`: pure presentation helpers for percentages and reset times.
- `src/App.tsx`: widget states and refresh interactions.
- `src/App.test.tsx`: UI-state tests.
- `src/lib/format.test.ts`: pure formatting tests.
