# Clodx 启动与状态栏设计

## 目标

Clodx 作为状态栏工具启动后静默驻留在 macOS 菜单栏；用户通过图标打开详情。开发时 `npm run tauri dev` 自动启动 Vite，不再要求手动启动两个进程。

## 正式版行为

- 主窗口启动时保持隐藏，不抢占焦点。
- 菜单栏标题优先显示 5 小时额度；若该窗口不存在则显示周额度，例如 `WK 97%`；两者皆不可用时显示 `C --`。
- 左键点击菜单栏图标显示或隐藏主窗口。
- 右键菜单保留“显示 / 隐藏”“立即刷新”“退出 Clodx”。

## 开发行为

- Tauri 配置的 `beforeDevCommand` 运行 `npm run dev -- --host 127.0.0.1`。
- `devUrl` 使用 `http://127.0.0.1:5173`，与开发服务器地址一致。
- 因此单独执行 `npm run tauri dev` 会完成前端服务和桌面程序启动。

## 数据与错误处理

- 刷新成功后，状态栏按 5H、WK 的优先级更新标题。
- 刷新失败时保留上一轮标题，避免把可用额度立刻替换为错误状态。
- 周额度是 7 天窗口（604800 秒）；它不会被标成 5H。

## 验证

- Rust 测试验证菜单栏标题对 5H、WK 与无数据三种情况的优先级。
- 前端与 Rust 全量测试、生产构建均通过。
- 手动运行 `npm run tauri dev`，确认不需要先手动运行 Vite。
